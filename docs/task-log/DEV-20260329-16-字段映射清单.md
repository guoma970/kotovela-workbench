# DEV-20260329-16 字段映射清单（开源 Playbook 对齐）

> 作用：为 `TSK-106`（GitHub 同步字段映射）提供“字段名 + 类型 + 来源 + 是否必填 + 映射规则”闭环清单。
> 数据源以 `GET /api/office-instances` 回包为主；当前实现位于 `src/data/officeInstancesAdapter.ts`。

## 1) 全量映射（Office 实例 -> 工作台内部模型）

> 所有输出字段均为工作台内部展示字段（`src/types/index.ts` 中 `Agent/Project/Task/Room`）。

### 1.1 Agent

| 输出字段 | 类型 | 是否必填 | 来源字段（按优先级） | 映射规则 |
|---|---|---|---|---|
| `id` | `string` | 是 | `item.key` | 先查 `item.key`，映射到固定实例码(`agent-1`..)，否则 `office-${key}`；无 key 则 `office-unknown`。 |
| `code` | `string` | 是 | `item.key` | 从 `OFFICE_AGENT_CODE_MAP` 命中则使用 `INS-xx`，否则 `key.toUpperCase()` 或 `INS-00`。 |
| `name` | `string` | 是 | `name` / 回退实例名 | `pickString(item.name)`；为空则回退到基线实例名/`实例 ${key}`。 |
| `role` | `string` | 是 | `role` | `pickString(item.role)`，为空回退 `未设置角色`。 |
| `status` | `"active" | "idle" | "blocked"` | 是 | `status` | 先 normalize 到内部语义（`doing/blocker/done/active/idle`）后再映射：`doing->active`, `blocker/blocked->blocked`, `done->idle`, `idle/unknown->idle`。 |
| `currentTask` | `string` | 是 | `currentTask` / `current_task` / `task` | 取第一个非空值；否则回退 `暂无任务`。 |
| `project` | `string` | 是 | `projectName` / `project` / `mainProject` / 默认 | 取第一个非空值；否则 `KOTOVELA`。 |
| `projectId` | `string` | 是 | `projectId` / `project_id` / fallback agent 的 projectId | 无可用值时回退 `project-1`。 |
| `updatedAt` | `string` | 是 | `ageText` / `updatedAt` / `updatedAtText` | 取第一个非空值；回退到实例基线值。 |

### 1.5 数据预处理规则（字段级）

- `normalizeStatus`：将状态词归一化，未知值 fallback `idle`（低风险，不抛错）。
- `pickString / pickFirst*`：统一处理 `string|number|unknown`。
- `parseInstanceArrayIds`：支持数组和逗号分隔字符串。
- `pickFirstIdFromValues`：支持 `id/key/taskId` 等嵌套字段。

## 3) 收口清单 unblock 最小行动（TSK-217 / TSK-106 / TSK-403）

### TSK-217（clawhub 目录结构）
- **最小行动**：形成 1 条群决议：`目录是否按项目线优先`（Owner：小树）。
- **验收**：在对应群消息里出现明确结论 + 命名标准（例如 `项目/章节/主题`）并用于后续更新。
- **安全依赖**：无（主要决策依赖业务拍板）。

### TSK-106（GitHub 字段映射）
- **最小行动**：补齐 `GitHub` 数据源字段映射到当前三层模型：
  1) `repo/full_name` -> `Project.name`；2)`issues` -> `Task.*`；3)`members` -> `Agent.*`。
  2) 给 `OfficeInstanceItem` 增加 `sourceType`/`repo` 等可选字段的类型声明与解析分支。
- **验收**：新增字段在同一份映射文档中有来源与规则；至少增加 1 个单元/手工验证步骤（返回样本 -> 映射后数据可展示）。

### TSK-403（演示素材与一言脚本）
- **最小行动**：建立固定模板（3–5 句）：产品背景/问题/进度/阻塞/下一步；素材仅留 1–2 张截图。
- **验收**：在开发群确认 1 套可用脚本（<60s）+ 素材清单（文件名/用途）。

## 4) 风险清单与缓解

| 风险 | 影响 | 缓解 |
|---|---|---|
| 源字段来源不稳定（多平台字段命名不一致） | 映射错误导致展示空值 | 保持多源键读取（如 `project_id` + `projectId`），关键字段加 schema 校验与告警埋点。 |
| 映射 fallback 过度（未知状态->idle） | blocker/异常被吞掉 | 对 unknown 状态记录 `debug` 日志，必要时将 `status` 显示为 `todo` + warning 标记。 |
| 项目/房间 ID 回退过于机械（`project-${key}`） | 跨页联动重复/错关联 | 在回退前尽量优先用 `projectName` 语义映射；并在发布前对回传样本做 `projectId` 采样校验。 |
| ID / code 中含特殊字符 | 路由/查询联动错位 | 统一 `trim` + 字符清洗（如空格替换），对关键 id 限制字符集合并记录修复。 |
| 未授权字段透传 | 隐私/安全风险 | 显式白名单选择字段，过滤未知对象；日志只记录字段名不输出敏感值。 |
